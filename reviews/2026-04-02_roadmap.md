- **`portfolio.py:record_entry()` の初期ストップが `price * 0.92` でハードコード**

## Phase 5: リアル移行準備（〜3ヶ月）

ペーパートレードで十分なエッジが確認できた場合。

### 5-1. 移行判断基準
以下をすべて満たすこと:
- [ ] 3ヶ月以上のペーパートレード実績
- [ ] プロフィットファクター > 1.3
- [ ] ウォークフォワード検証で正のリターン
- [ ] 最大ドローダウンが許容範囲内（例: -15% 以下）

### 5-2. 証券 API 連携
- [ ] S株対応の証券 API 調査（SBI証券 API / kabu STATION 等）
- [ ] 注文実行モジュール（エントリー・イグジット・部分利確）
- [ ] リアルタイム約定確認
- [ ] ペーパーモードとの切り替え（config.yaml の `mode: real`）

### 5-3. 安全装置
- [ ] 日次損失リミット（例: -3% で当日の新規エントリー停止）
- [ ] 手動オーバーライド（Slack から緊急全ポジション売却）
- [ ] 注文の二重送信防止
- [ ] リアルモード開始時の確認プロンプト

### 5-4. 運用体制
- [ ] trades.json のクラウドバックアップ（S3 / Google Drive）
- [ ] 実行ログの長期保存（30日ローテーション → 1年アーカイブ）
- [ ] 月次パフォーマンスレポートの自動生成
